# Support

For usage questions and reproducible bugs, open a GitHub issue and include:

- Operating system, Docker version, and browser.
- The exact prompt or workflow.
- Relevant ticker, interval, dataset, and strategy names.
- Sanitized backend logs and the agent run ID, if available.
- The selected LLM provider and model, without including credentials.

Before opening an issue, pull the latest code, rebuild with `docker compose up -d --build`, and check the existing issues.

Do not report vulnerabilities or credential leaks publicly. Follow [SECURITY.md](SECURITY.md).
